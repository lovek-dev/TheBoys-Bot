Core Requirements
Use slash commands only
Use modular command + event structure
Store persistent data using JSON or SQLite (preferred)
Include proper error handling and logging
Use environment variables for token and config
🎬 Feature Set
1. Watch Party System

Command: /startmovie <name>

Fetch movie details from an API (OMDb or TMDB)
Send embed with:
Title, rating, duration, genre, plot
Create:
A thread for discussion
Optional:
Countdown timer (5 min default)
2. Reaction Tracking System
Provide buttons:
😂 Funny
😱 Scary
🤯 Plot twist
💀 Cringe
Store timestamps + counts per movie session
Command: /movieresults
Show most reacted moments
3. Recommendation System

Command: /recommend

Use stored ratings/history
Suggest movies based on:
genre overlap
Simple logic is fine (no ML required)
4. Rating System

Command: /rate <movie> <1-10> [review]

Store user ratings
Command: /ratings
Show average rating + top review
5. Trivia System

Command: /trivia

Ask movie-related questions
Reward points
Store leaderboard
6. Movie Voting System

Command: /pollmovie

Users suggest movies
Bot creates voting embed
After timeout → announce winner
7. Resume System

Command: /marktime <hh:mm:ss>
Command: /resume

Save and display last timestamp
8. Movie Roles System
Allow tagging roles for movie alerts
9. Reminder System

Command: /remindmovie <time>

Sends scheduled reminders
10. Stats Dashboard

Command: /moviestats

Show:
total movies watched
most active user
top rated movie
🔒 GLOBAL TOGGLE SYSTEM (IMPORTANT)
Command: /movieserver
This command should:
Ask: "Enable or Disable movie features?"
Options: enable, disable
Permissions:
ONLY usable by the bot owner

Owner ID will be stored in config file:

OWNER_ID=your_id_here
Behavior:

Store server-specific state:

guildId → enabled: true/false
If disabled:
ALL movie-related commands should NOT work
Bot should reply:
"Movie features are disabled in this server."
If enabled:
All features function normally
This must be checked in a global command handler middleware
